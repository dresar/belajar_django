Alur Kerja Model-View-Template (MVT)

Pola MVT adalah jantung arsitektur Django. Diagram interaktif di bawah ini menggambarkan bagaimana permintaan web mengalir melalui komponen-komponen ini untuk menghasilkan respons. Klik setiap komponen untuk melihat peran spesifiknya.

Klik salah satu kotak di atas untuk melihat penjelasan detail tentang perannya dalam alur MVT.

Browser Pengguna (Request): Inilah titik awal! Pengguna mengirimkan permintaan (misalnya, mengetik URL, mengklik tautan) ke server web Django Anda.

URL Dispatcher (urls.py): Django menerima permintaan dan mencocokkan URL dengan pola yang Anda definisikan. Jika cocok, ia meneruskan permintaan ke view yang sesuai.

View (views.py): Ini adalah fungsi atau kelas Python yang menerima permintaan, berisi logika bisnis. Ia dapat berinteraksi dengan Model (untuk data) dan/atau Template (untuk presentasi) sebelum mengembalikan respons.

Model (models.py): Ini adalah representasi data Anda. Kelas Python yang mendefinisikan struktur database Anda. View menggunakan Model untuk mengambil, menyimpan, dan memanipulasi data melalui ORM Django.

Template (.html): Ini adalah file HTML dengan sintaks khusus Django. Template bertanggung jawab untuk merender data yang diterima dari View menjadi HTML akhir yang dikirim kembali ke pengguna.

Database: Tempat data disimpan secara persisten. Model Django berkomunikasi dengan database menggunakan ORM (Object-Relational Mapper) sehingga Anda tidak perlu menulis SQL.

Models: Definisi & Tipe Field

Models adalah inti dari setiap aplikasi Django, mendefinisikan struktur data Anda dan menyediakan antarmuka untuk berinteraksi dengan database. Pelajari cara mendefinisikan model dan berbagai tipe field yang tersedia.

Definisi Model Dasar â–¼

Model Django adalah kelas Python yang mewarisi dari django.db.models.Model. Setiap atribut kelas ini menjadi kolom (field) di tabel database.

# blog/models.py
from django.db import models

class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    pub_date = models.DateTimeField(auto_now_add=True)
    is_published = models.BooleanField(default=False)

    def __str__(self):
        return self.title

    class Meta:
        ordering = ['-pub_date']

__str__ adalah metode penting yang menentukan representasi string dari objek model, sangat berguna di situs admin Django. Meta class digunakan untuk memberikan opsi-opsi pada model, seperti urutan default.

Tipe Field Umum â–¼

Django menyediakan berbagai tipe field untuk merepresentasikan berbagai jenis data:

  • CharField(max_length=...): Untuk string pendek hingga menengah.
  • TextField(): Untuk teks panjang.
  • IntegerField(): Untuk bilangan bulat.
  • BooleanField(default=...): Untuk nilai true/false.
  • DateTimeField(auto_now_add=True/auto_now=True): Untuk tanggal dan waktu.
  • DateField(): Untuk tanggal saja.
  • EmailField(): Untuk alamat email.
  • URLField(): Untuk URL.
  • DecimalField(max_digits=..., decimal_places=...): Untuk angka desimal presisi tinggi.
  • ImageField(upload_to=...): Untuk menyimpan jalur file gambar.
  • FileField(upload_to=...): Untuk menyimpan jalur file umum.

Models: Hubungan & QuerySet API

Hubungan antar model adalah fundamental dalam database relasional. Django ORM (Object-Relational Mapper) membuat interaksi dengan database menjadi intuitif menggunakan Python murni melalui QuerySet API.

Hubungan Antar Model â–¼

Django mendukung tiga jenis hubungan database:

  • One-to-Many (ForeignKey): Satu record di satu tabel dapat dihubungkan dengan banyak record di tabel lain. Contoh: Satu Penulis memiliki banyak Post.
    • from django.db import models

class Author(models.Model):
    name = models.CharField(max_length=100)

class Post(models.Model):
    title = models.CharField(max_length=200)
    author = models.ForeignKey(Author, on_delete=models.CASCADE) # One-to-Many
  • Many-to-Many (ManyToManyField): Banyak record di satu tabel dapat dihubungkan dengan banyak record di tabel lain. Contoh: Satu Post memiliki banyak Tag, satu Tag memiliki banyak Post.
    • from django.db import models

class Tag(models.Model):
    name = models.CharField(max_length=50)

class Post(models.Model):
    # ... other fields
    tags = models.ManyToManyField(Tag) # Many-to-Many
  • One-to-One (OneToOneField): Satu record di satu tabel hanya dapat dihubungkan dengan satu record di tabel lain. Contoh: Satu Pengguna memiliki satu Profil Pengguna.
    • from django.db import models
from django.contrib.auth.models import User

class UserProfile(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE)
    bio = models.TextField(blank=True)
QuerySet API: Mengambil Data â–¼

QuerySet API adalah cara Anda berinteraksi dengan database menggunakan model Django:

  • Model.objects.all(): Mengambil semua objek.
  • Model.objects.get(id=1): Mengambil satu objek berdasarkan kriteria. Akan memunculkan error jika tidak ditemukan atau ditemukan lebih dari satu.
  • Model.objects.filter(field='value'): Mengambil QuerySet yang cocok dengan kriteria.
  • Model.objects.exclude(field='value'): Mengambil QuerySet yang tidak cocok dengan kriteria.
  • QuerySet.order_by('field'): Mengurutkan hasil.
  • QuerySet.count(): Menghitung jumlah objek.
  • QuerySet.first() / QuerySet.last(): Mengambil objek pertama/terakhir.
# Contoh penggunaan di Django shell atau view
from blog.models import Post, Author

# Membuat data
author = Author.objects.create(name='John Doe')
Post.objects.create(title='Django Basics', author=author, content='Content here.')
Post.objects.create(title='Advanced Django', author=author, content='More content.', is_published=True)

# Mengambil semua post
all_posts = Post.objects.all()

# Mengambil post yang diterbitkan
published_posts = Post.objects.filter(is_published=True)

# Mengambil post berdasarkan ID
post_by_id = Post.objects.get(id=1)

# Mengambil post dari penulis tertentu
john_posts = Post.objects.filter(author__name='John Doe')

# Mengambil post dan mengurutkan
ordered_posts = Post.objects.order_by('-pub_date')
QuerySet API: Membuat, Memperbarui, Menghapus â–¼
  • Membuat: 
    # Cara 1: Menggunakan create()
post = Post.objects.create(title="Judul Baru", content="Isi tulisan.")

# Cara 2: Membuat objek dan menyimpan
new_post = Post(title="Judul Lain", content="Isi lainnya.")
new_post.save()
  • Memperbarui: 
    # Memperbarui satu objek
post = Post.objects.get(id=1)
post.title = "Judul Diperbarui"
post.save()

# Memperbarui banyak objek sekaligus
Post.objects.filter(is_published=False).update(is_published=True)
  • Menghapus: 
    # Menghapus satu objek
post = Post.objects.get(id=1)
post.delete()

# Menghapus banyak objek
Post.objects.filter(pub_date__year=2023).delete()
Migrasi Database â–¼

Migrasi adalah cara Django menyebarkan perubahan yang Anda buat pada model (menambahkan field, menghapus model, dll.) ke skema database Anda.

  • python manage.py makemigrations [app_name]: Membuat file migrasi baru berdasarkan perubahan pada model Anda.
  • python manage.py migrate: Menerapkan migrasi yang belum diterapkan ke database.
  • python manage.py showmigrations [app_name]: Menampilkan daftar semua migrasi dan statusnya.

Views: Function-Based (FBV) vs. Class-Based (CBV)

Views adalah "otak" aplikasi Django Anda, bertanggung jawab untuk memproses permintaan web dan mengembalikan respons. Ada dua cara utama untuk menulis views: sebagai fungsi Python (FBV) atau sebagai kelas Python (CBV).

Function-Based Views (FBV) â–¼

FBV adalah fungsi Python sederhana yang menerima objek HttpRequest dan mengembalikan objek HttpResponse. Mereka mudah dipahami dan cepat untuk ditulis.

# blog/views.py (FBV Contoh)
from django.shortcuts import render, redirect, get_object_or_404
from .models import Post

def post_list(request):
    posts = Post.objects.all().order_by('-pub_date')
    return render(request, 'blog/post_list.html', {'posts': posts})

def post_detail(request, pk):
    post = get_object_or_404(Post, pk=pk)
    return render(request, 'blog/post_detail.html', {'post': post})

def add_post(request):
    if request.method == 'POST':
        # Proses form data
        pass
    return render(request, 'blog/add_post.html')
Class-Based Views (CBV) â–¼

CBV adalah kelas Python yang mewarisi dari kelas dasar generic view Django. Mereka menyediakan struktur yang lebih terorganisir untuk kode yang dapat digunakan kembali dan memungkinkan modularitas yang lebih baik. Cocok untuk operasi CRUD (Create, Retrieve, Update, Delete) umum.

# blog/views.py (CBV Contoh)
from django.views.generic import ListView, DetailView, CreateView, UpdateView, DeleteView
from .models import Post
from django.urls import reverse_lazy

class PostListView(ListView):
    model = Post
    template_name = 'blog/post_list.html'
    context_object_name = 'posts'
    ordering = ['-pub_date']

class PostDetailView(DetailView):
    model = Post
    template_name = 'blog/post_detail.html'
    context_object_name = 'post'

class PostCreateView(CreateView):
    model = Post
    template_name = 'blog/add_post.html'
    fields = ['title', 'content', 'author', 'is_published']
    success_url = reverse_lazy('blog:list')
Objek HttpRequest & HttpResponse â–¼

Setiap *view* menerima objek HttpRequest dan harus mengembalikan objek HttpResponse.

# Contoh sederhana
from django.http import HttpResponse

def simple_view(request):
    if request.method == 'GET':
        return HttpResponse("Ini adalah respons GET.")
    elif request.method == 'POST':
        return HttpResponse("Ini adalah respons POST.")

URLs: Pemetaan Lanjutan

URL dispatcher Django adalah komponen kunci yang merutekan permintaan masuk ke views yang benar. Pelajari cara menggunakan pemetaan URL yang lebih canggih, termasuk parameter dinamis dan namespace.

Penggunaan path() Lanjutan â–¼

Fungsi path() menerima beberapa argumen untuk fleksibilitas:

  • route: String yang berisi pola URL.
  • view: Fungsi atau kelas view yang akan dipanggil.
  • kwargs: (Opsional) Kamus argumen kata kunci yang akan diteruskan ke view.
  • name: (Opsional) Nama untuk pola URL, digunakan untuk referensi terprogram (misalnya, {% url %} di template, reverse() di Python).
# blog/urls.py
from django.urls import path
from . import views

app_name = 'blog'
urlpatterns = [
    path('', views.PostListView.as_view(), name='list'),
    path('post/<int:pk>/', views.PostDetailView.as_view(), name='detail'),
    path('post/new/', views.PostCreateView.as_view(), name='create'),
    path('about/', views.about_page, {'template_name': 'about.html'}, name='about'), # Dengan kwargs
]
Regex dengan re_path() â–¼

Untuk pola URL yang lebih kompleks yang memerlukan ekspresi reguler (regex), gunakan re_path(). Ini dulunya adalah cara utama pemetaan URL sebelum path() diperkenalkan.

# blog/urls.py (menggunakan re_path)
from django.urls import re_path
from . import views

urlpatterns = [
    re_path(r'^articles/(?P<year>[0-9]{4})/$', views.year_archive),
]
URL Namespaces â–¼

Namespace mencegah konflik nama URL antar aplikasi. Anda bisa memiliki detail URL di aplikasi `blog` dan `portfolio` tanpa kebingungan.

# myproject/urls.py
from django.urls import path, include

urlpatterns = [
    path('blog/', include('blog.urls', namespace='blog')),
    path('portfolio/', include('portfolio.urls', namespace='portfolio')),
]

# Di template atau Python:
# {% url 'blog:detail' post.id %}
# {% url 'portfolio:project_detail' project.id %}

Templates: Pewarisan & Fitur Lanjutan

Django Template Language (DTL) adalah bahasa template sederhana yang berfokus pada pemisahan logika dari presentasi. Pelajari bagaimana pewarisan template dan berbagai tag/filter meningkatkan efisiensi pengembangan.

Pewarisan Template (Inheritance) â–¼

Pewarisan template memungkinkan Anda untuk membuat layout dasar sekali dan menggunakannya kembali di seluruh situs Anda, mengurangi duplikasi kode. Gunakan tag {% extends %} dan {% block %}.

<!-- templates/base.html -->
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}Situs Saya{% endblock %}</title>
</head>
<body>
    <div id="content">
        {% block content %}</div>
    {% endblock %}
</body>
</html>

<!-- blog/templates/blog/post_list.html -->
{% extends 'base.html' %}

{% block title %}Daftar Post Blog{% endblock %}

{% block content %}
    <h1>Semua Post</h1>
    <ul>
        {% for post in posts %}
            <li>{{ post.title }}</li>
        {% endfor %}
    </ul>
{% endblock %}
Template Tags & Filters Umum â–¼

Tags ({% ... %}): Melakukan logika seperti perulangan, kondisi, memuat file statis.

  • {% for item in list %} ... {% endfor %}
  • {% if condition %} ... {% else %} ... {% endif %}
  • {% load static %}: Memuat tag untuk file statis.
  • {% csrf_token %}: Penting untuk keamanan form POST.
  • {% url 'app_name:url_name' arg1 arg2 %}: Membuat URL.

Filters ({{ value|filter_name:argument }}): Memodifikasi output variabel.

  • {{ value|date:"Y-m-d" }}: Memformat tanggal.
  • {{ value|length }}: Mengembalikan panjang string/list.
  • {{ value|lower }} / {{ value|upper }}: Mengubah kasus teks.
  • {{ value|truncatewords:num }}: Memotong teks setelah sejumlah kata.

Static Files (CSS, JS, Images)

Django menyediakan cara terorganisir untuk mengelola file statis seperti CSS, JavaScript, dan gambar. Pelajari cara mengaturnya untuk pengembangan dan *deployment*.

Konfigurasi Dasar â–¼

Dalam settings.py Anda:

# myproject/settings.py
import os

STATIC_URL = '/static/'

# Direktori tempat Django akan mencari file statis di luar app Anda
STATICFILES_DIRS = [
    os.path.join(BASE_DIR, 'static'),
]

# Direktori tempat file statis akan dikumpulkan untuk deployment
STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')

Pastikan Anda memiliki folder static di root project Anda, dan di dalam setiap app Anda.

Menggunakan Static Files di Template â–¼

Untuk menggunakan file statis di template Anda, pertama-tama muat tag static, lalu gunakan tag {% static %}.

<!-- templates/base.html atau app_template.html -->
{% load static %}
<!DOCTYPE html>
<html>
<head>
    <link rel="stylesheet" href="{% static 'css/style.css' %}">
    <script src="{% static 'js/main.js' %}"></script>
</head>
<body>
    <img src="{% static 'images/logo.png' %}" alt="Logo">
</body>
</html>
Mengumpulkan Static Files untuk Deployment â–¼

Saat *deployment*, Anda perlu mengumpulkan semua file statis dari berbagai lokasi (aplikasi, STATICFILES_DIRS) ke satu direktori yang ditentukan oleh STATIC_ROOT. Server web kemudian akan menyajikan file-file ini.

python manage.py collectstatic

Forms Dasar

Django Forms menyederhanakan proses penanganan input pengguna, dari validasi hingga rendering. Pelajari dasar-dasar membuat dan memproses form di aplikasi Django Anda.

Membuat Form â–¼

Definisikan form Anda dalam file forms.py:

# blog/forms.py
from django import forms

class PostForm(forms.Form):
    title = forms.CharField(max_length=200, label="Judul Post")
    content = forms.CharField(widget=forms.Textarea, label="Konten")
    is_published = forms.BooleanField(required=False, label="Terbitkan?")

# Anda juga bisa menggunakan ModelForm untuk membuat form dari model:
# from django.forms import ModelForm
# from .models import Post

# class PostModelForm(ModelForm):
#     class Meta:
#         model = Post
#         fields = ['title', 'content', 'is_published']
Menangani Form di View â–¼

Proses pengiriman form biasanya melibatkan memeriksa metode permintaan, memvalidasi data, dan menyimpannya.

# blog/views.py
from django.shortcuts import render, redirect
from .forms import PostForm
from .models import Post # Jika menggunakan ModelForm

def create_post(request):
    if request.method == 'POST':
        form = PostForm(request.POST) # PostForm(request.POST) untuk ModelForm
        if form.is_valid():
            # Jika menggunakan forms.Form
            title = form.cleaned_data['title']
            content = form.cleaned_data['content']
            is_published = form.cleaned_data['is_published']
            Post.objects.create(title=title, content=content, is_published=is_published)
            
            # Jika menggunakan ModelForm:
            # form.save() 
            return redirect('blog:list')
    else:
        form = PostForm() # Instansiasi form kosong untuk permintaan GET
    return render(request, 'blog/post_form.html', {'form': form})

Situs Admin Kustomisasi

Situs admin Django adalah fitur yang sangat kuat untuk mengelola data Anda. Pelajari cara menyesuaikannya agar lebih ramah pengguna dan sesuai dengan kebutuhan Anda.

Mendaftarkan Model â–¼

Untuk membuat model Anda tersedia di situs admin, daftarkan di admin.py app Anda.

# blog/admin.py
from django.contrib import admin
from .models import Post

admin.site.register(Post)
Menggunakan ModelAdmin â–¼

Untuk kustomisasi lebih lanjut, buat kelas ModelAdmin dan daftarkan model Anda dengannya.

  • list_display: Field yang akan ditampilkan di daftar objek.
  • list_filter: Field yang dapat difilter di sidebar.
  • search_fields: Field yang dapat dicari.
  • list_per_page: Jumlah item per halaman.
  • fieldsets: Mengatur field ke dalam grup visual.
# blog/admin.py
from django.contrib import admin
from .models import Post, Author

class PostAdmin(admin.ModelAdmin):
    list_display = ('title', 'author', 'pub_date', 'is_published')
    list_filter = ('is_published', 'pub_date', 'author')
    search_fields = ('title', 'content')
    date_hierarchy = 'pub_date'
    ordering = ('-pub_date',)
    
    fieldsets = (
        (None, {
            'fields': ('title', 'content')
        }),
        ('Informasi Publikasi', {
            'fields': ('author', 'pub_date', 'is_published'),
            'classes': ('collapse',) # Buat bagian ini bisa dilipat
        }),
    )

admin.site.register(Post, PostAdmin)
admin.site.register(Author)

Otentikasi Dasar

Django hadir dengan sistem otentikasi dan otorisasi yang tangguh. Pelajari cara menggunakan model pengguna, melindungi views, dan memeriksa status otentikasi.

Menggunakan Model User â–¼

Model User bawaan Django dapat diakses melalui django.contrib.auth.models.User. Anda dapat menggunakannya seperti model lainnya.

from django.contrib.auth.models import User

# Membuat user baru
user = User.objects.create_user('john_doe', 'john@example.com', 'password123')

# Mencari user
try:
    user = User.objects.get(username='john_doe')
except User.DoesNotExist:
    print("User tidak ditemukan.")

# Mengecek properti user di view/template
# request.user.is_authenticated
# request.user.is_staff
# request.user.is_superuser
Melindungi Views dengan Decorator â–¼

Gunakan decorator @login_required untuk memastikan hanya pengguna yang *login* yang dapat mengakses *view* tertentu.

# blog/views.py
from django.contrib.auth.decorators import login_required
from django.shortcuts import render

@login_required
def my_secret_page(request):
    return render(request, 'blog/secret_page.html', {'user': request.user})

# Untuk Class-Based Views, gunakan mixin:
# from django.contrib.auth.mixins import LoginRequiredMixin
# from django.views.generic import TemplateView

# class SecretPageView(LoginRequiredMixin, TemplateView):
#     template_name = 'blog/secret_page.html'

Anda perlu mengkonfigurasi LOGIN_URL di settings.py:

# myproject/settings.py
LOGIN_URL = '/accounts/login/' # Ganti dengan URL login Anda
